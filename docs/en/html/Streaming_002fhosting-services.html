Other useful information.

For the following hosting services it is recommended to extend the maximum download limit (add the -m option to the instructions listed above, possibly indicating the maximum number of downloads to be made simultaneously to proceed with the download in parallel, also through the use of proxies

Many servers do not allow recovery of interrupted downloads and parallel downloading. For these services ZDL uses Wget.

Some services require the use of RTMPDump (in its absence, but with limitations, of cURL)

The other services are enabled by default for Aria2.

To download from Sharpfile, Depositfiles and other services, currently it is necessary to use the web browser to solve the captcha, generate the link and start the download through the browsers adds-on or with the ZDL booted from the terminal.

The --login option enables automatic login in the event of the ZDL configuration includes one or more accounts for the use of a hosting service. In particular, the registration of many accounts for Easybytez not only allows the download of files up to size at 600MB, but also allows multiple files to be downloaded in parallel without the use of proxies (in this case it is also necessary to specify -m or --max-downloads=). To configure the use of accounts: zdl [-c|--configure]

For each service or link, should Aria2 fail to connect, try using Axel or Wget.
